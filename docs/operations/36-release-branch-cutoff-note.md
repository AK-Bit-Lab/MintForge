# Release branch cutoff note

Define a clear cutoff time for release branch changes.
Late changes should be explicitly re-approved with updated smoke evidence.

Write down the cutoff timestamp, active branch protections, and merge queue state before release handoff begins.

Pin the cutoff commit hash in the release thread so branch context stays explicit.

Mark the cutoff timestamp in UTC in both release issue and chat to avoid timezone confusion.

Escalate late cutoff updates that leave less than one full review cycle before branch freeze.

Verification evidence: include a release timeline note showing when cutoff notice was shared.

Follow-up cadence: confirm branch cutoff communication one day and one hour before freeze.

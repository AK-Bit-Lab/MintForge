# Incident thread template

Use a fixed incident thread template with impact, timeline, and owner sections.
Consistent structure improves response coordination under pressure.

Start every incident thread with impact scope, owner, last-known-good release metadata, and current mitigation status.

Include first detection time and user-impact estimate in the opening incident update.

Include a direct link to the incident chat thread in every template use so context is preserved for follow-up.

Escalate missing incident timeline entries when status changes cannot be correlated to mitigation actions.

Verification evidence: include a completed incident thread sample with timestamps for each required section.

Follow-up cadence: review incident template completeness at the end of each on-call week.
